========================================
한방진단 (Med-Herb) 포트폴리오
========================================


1) 포트폴리오 소개
----------------------------------------

[서비스 카테고리]
헬스케어 / 한의학 자가진단 서비스

[메인 타깃]
건강에 관심 있는 30-50대 일반인

[서비스 소개]
한방진단은 사용자가 자신의 증상을 입력하면 한의학적 변증(체질 경향)을
도출하고, 그에 맞는 치료 방향과 약재 정보를 제공하는 웹 서비스입니다.

일반인이 어렵게 느끼는 한의학 정보(기허, 담음, 어혈 등)를
간단한 설문을 통해 쉽게 이해할 수 있도록 도와주며,
결과에 대한 문헌 근거를 함께 제공하여 신뢰성을 확보합니다.


2) 작업 범위
----------------------------------------

[개발 참여 범위]
풀스택 개발 (Frontend + Backend + Database + Admin)

[상세 범위]
- Frontend: React 기반 SPA 전체 개발 (사용자 페이지 + 관리자 페이지)
- Backend: Cloudflare Workers + Hono 프레임워크 API 서버 개발
- Database: Cloudflare D1 (SQLite) 스키마 설계 및 마이그레이션
- Admin: 관리자 인증 및 데이터 CRUD 페이지
- Testing: 단위 테스트 (Vitest) + E2E 테스트 (Playwright)
- Deployment: Cloudflare Pages + Workers 배포

[지원 환경]
- 플랫폼: PC 웹 전용 (Desktop)
- 브라우저: Chrome, Firefox, Safari, Edge 최신 버전

[기술 스택]
- Frontend: React 19, TypeScript, Vite, TailwindCSS, Zustand, TanStack Query, Axios
- Backend: Cloudflare Workers, Hono, TypeScript, Drizzle ORM, Zod
- Database: Cloudflare D1 (SQLite)
- Testing: Vitest, Playwright, MSW
- Deployment: Cloudflare Pages, Cloudflare Workers


3) 주요 업무
----------------------------------------

[주요 기능]

1. 증상 진단 설문 (FEAT-1)
   - 증상 체크박스 선택 (두통, 복통, 피로, 소화불량 등)
   - 상황 질문 10~15개 응답 (라디오/슬라이더 방식)
   - 진행률 표시로 완료까지 동기 부여

2. 변증 결과 표시 (FEAT-2)
   - 한의학적 변증 후보 3개 카드 형태 제공
   - 각 변증별 매칭 점수/경향도 표시
   - 결과 도출 근거 명시 (bullet 2~3개)

3. 치료축/약재 추천 (FEAT-3)
   - 공통 치료 접근 축 3개 (예: 보기, 이기, 활혈)
   - 추천 약재 5개 테이블 (이름, 효능, 근거)
   - 문헌 근거 링크 제공

4. 관리자 기능 (FEAT-0)
   - JWT 기반 관리자 로그인/인증
   - 증상 데이터 CRUD
   - 질문 데이터 CRUD
   - 변증 데이터 CRUD
   - 약재 데이터 CRUD

[주요 페이지]

- 사용자 페이지
  - 메인/랜딩 페이지
  - 증상 선택 페이지
  - 상황 질문 페이지
  - 변증 결과 페이지
  - 치료축/약재 추천 페이지

- 관리자 페이지
  - 로그인 페이지
  - 대시보드
  - 증상 관리
  - 질문 관리
  - 변증 관리
  - 약재 관리


4) 주안점
----------------------------------------

[1. 진단 완료율 70% 목표 달성]
- 노스스타 지표: 설문 시작 후 결과 확인까지 완료한 비율
- 질문 수 10~15개로 제한 (기존 서비스의 "질문이 너무 많음" 문제 해결)
- 모든 단계에 진행률 표시로 이탈 방지
- 2단계 결과 화면(변증 → 약재) 구조로 정보 과부하 방지

[2. 성능 최적화]
- 목표: 진단 결과 산출 3초 이내
- 실제 성능: 평균 0.51초 달성 (목표 대비 83% 개선)
- Cloudflare Edge Network 활용으로 글로벌 응답 속도 확보
- API 응답 P95 < 500ms 달성

[3. 보안 강화]
- JWT 기반 관리자 인증 (Access Token + Refresh Token)
- PBKDF2 비밀번호 해싱
- HTTPS 필수 적용 (Cloudflare 기본 제공)
- Zod 스키마 기반 서버 측 입력 검증
- CORS 허용 도메인 제한

[4. Contract-First TDD 개발 방식]
- API 계약(Contract) 선 정의 후 BE/FE 독립 병렬 개발
- 테스트 코드 선작성 (RED → GREEN → REFACTOR 사이클)
- MSW를 활용한 Frontend 독립 개발 환경 구축
- 단위 테스트 + E2E 테스트로 품질 보장

[5. 사용자 신뢰 확보]
- 모든 결과에 문헌 근거 명시
- 왜 이런 결과가 나왔는지 투명하게 설명
- 변증 후보 3개를 제시하여 가능성 범위 안내 (단정적 진단 지양)
- 깔끔하고 신뢰감 있는 UI 디자인 적용
