# 한의학 증상 진단 및 처방 로직 설문 시스템

## 프로젝트 개요

한의학 증상 진단 및 처방 로직 설문 시스템 구축

## 프로젝트 배경 및 목표

본 프로젝트는 상용 서비스 개발 전 시장 검증을 위한 MVP 개발 프로젝트입니다.

사용자가 입력한 증상 데이터를 바탕으로 한의학적 변증을 도출하고, 이에 따른 치료법과 약재 정보를 제공하는 웹 기반 시스템을 구축하는 것을 목표로 합니다. 진단 로직을 설문 형태로 자동화하여 사용자에게 직관적인 결과를 제시합니다.

## 과업 범위

### 1. 수행 범위

| 영역 | 비고 |
|------|------|
| UI/UX | 생성형 AI 또는 상용 템플릿 구입으로 대체 |
| 프런트엔드/Client 개발 | - |
| 백엔드 개발 | - |
| 서버/DB/인프라 구성 | - |

### 2. 상세 기능 요구 사항

#### 2-1. 진단 입력 모듈

- **입력 화면** (증상 + 상황 질문)
  - 증상 선택 (체크박스 5개, 예: 두통, 복통)
  - 상황 질문 10~15문항 (라디오/슬라이더)
  - "결과 보기" CTA 버튼

#### 2-2. 결과 도출 화면 1: 변증 후보 "지도"

- 변증 후보 카드 3개 (예: "기허 경향")
- "왜 이렇게 나왔는지" 2~3개 근거 bullet
- "근거 보기" 버튼

#### 2-3. 결과 화면 2: 교집합 치료축/약재 + 근거

- "공통으로 자주 등장하는 치료 접근 축" 3개
- 공통 약재 5개 (표 형태)
- 각 항목에 "근거" 링크 (문헌/논문)

#### 2-4. 로직 기반 설문 결과값 도출

- 사용자 입력 데이터 기반 결과화면 도출
- 설문에 필요한 로직은 클라이언트로부터 제공될 예정

### 3. 비기능적 요구사항

#### 3-1. 성능

- 진단 로직 처리 및 결과 산출 속도 최적화
- 동적 폼 전환 시 지연 없는 UI 반응성 확보

#### 3-2. 보안

- 관리자 페이지 접근 제어 및 인증 구현
- 사용자 입력 데이터의 기본적인 전송 보안 (HTTPS) 적용

#### 3-3. 데이터

- 진단 로직 및 질문 데이터의 구조화 (JSON 등)를 통한 확장성 고려
